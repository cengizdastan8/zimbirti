# TekPanel UI Principles

## Screen Hierarchy

1. App identity: TekPanel
2. Current inbox count
3. Channel control
4. Clear-screen action
5. Message list
6. Empty state

## Message Row Anatomy

Each message row should include:

- source badge
- sender name
- source app label
- capture method
- received time
- message text
- Okundu action
- optional package metadata

## Channel Panel Anatomy

Each channel row should include:

- platform/source badge
- channel label, such as WhatsApp Business
- package name as secondary technical text
- open/closed state

## Good Mobile Inbox Feel

- rows are compact but not cramped
- typography is strong enough for quick reading
- separators are visible but soft
- source color is clear without overwhelming the text
- actions are reachable and predictable

## Bad TekPanel UI

- dashboard metrics as the main focus
- fake CRM forms
- large empty cards that waste the viewport
- decorative background effects
- repeated identical cards with no channel identity
- buttons that clip at 390px
